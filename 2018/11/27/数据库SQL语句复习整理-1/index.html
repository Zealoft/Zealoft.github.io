定义、删除模式SCHEMA

定义模式

​ 例：定义一个学生-课程模式S-T

CREATE SCHEMA "S-T" AUTHORIZATION WANG;
/* 为用户WANG定义了一个模式S-T */

​ 或者，只指明用户的名称，此时模式名会隐含为用户名。

CREATE SCHEMA AUTHORIZATION WANG;

​ 实际上，定义模式就是定义了一个命名空间，在这个空间中可以定义该模式包含的数据库对象，例如基本表、视图、索引等。因此，可以在创建模式的同时在其中定义表、视图等等。

CREATE SCHEMA TEST AUTHORIZATION ZHANG
    CREATE TABLE TAB1(
        COL1 SMALLINT,
        COL2 INT,
        COL3 CHAR(20),
        COL4 NUMERIC(10, 3),
        COL5 DECIMAL(5,2)
    );

删除模式

​ 删除模式的一般语句是

DROP SCHEMA <模式名><CASCADE | RESTRICT>

​ 删除模式有两种类型分别是CASCADE和RESTRICE。

​ 其中CASCADE代表级联模式，也就是在删除模式的同时把该模式中的所有数据库对象全部删除；

​ 而RESTRICT则是限制模式，如果该模式中定义了下属的数据库对象（如表、视图等），则拒绝该删除语句的执行，也就是说只有在该模式中没有任何下属的对象时才可以执行。

对于表的操作(定义、删除、修改)​

SQL中的数据类型

​ 在对表进行操作之前，先来看一看SQL中都有哪些数据类型可供使用。

数据类型 含义
CHAR(n) 长度为n的定长字符串
VARCHAR(n) 最大长度为n的变长字符串
INT 长整数(也可以写为INTEGER)
SMALLINT 短整数
NUMERIC(p, d) 定点数，由p位数字组成，小数后面有d位数字
REAL 取决于机器精度的浮点数
Double Precision 取决于机器精度的双精度浮点数
FLOAT(n) 浮点数，精度至少为n位数字
DECIMAL(a, b) a指定小数点左右可以存储的十进制数字的最大个数，b指定小数点右边数字位数
DATE 日期，格式为YYYY-MM-DD
TIME 时间，包含时、分、秒，格式为HH:MM:SS

定义基本表

​ 基本表定义的一般格式为

CREATE TABLE <表名>
    (<列名><数据类型>[<列级完整性约束条件>]
    [, <列名><数据类型>[<列级完整性约束条件>]]...
    [, <表级完整性约束条件>]);

​ 其中，如果完整性约束条件涉及到该表的多个属性列，则必须定义在表级上(比如一个表的主码有多个列)，否则可以定义在列级也可以定义在表级。

​ 举个例子，建立表Student，学号是主码，姓名取值唯一。

CREATE TABLE Student
    (Sno CHAR(9) PRIMARY KEY, /* 列级完整性约束条件*/
     Sname CHAR(20) UNIQUE, /* Sname取唯一值*/
     Ssex CHAR(2),
     Sage smallint,
     Sdept char(20)
    );

​ 可以注意到其中的字符串既可以写成CHAR也可以写成char，是因为SQL语句中的关键字都是大小写不敏感的，个人比较喜欢使用全小写的方式声明关键字。

​ 再举一个前面提到的完整性约束条件涉及到该表的多个属性列的情况。

create table SC
    (Sno char(9),
     Cno char(4),
     Grade smallint,
     primary key(Sno, Cno),
     /* 主码由两个属性构成，必须作为表级完整性进行定义 */
     foreign key(Sno) references Student(Sno),
     /* 表级完整性约束条件，Sno是外码，被参照表是Student */
     foreign key(Cno) references Course(CNo)
    );

​ 由于SC表的主码有Sno和Cno两个属性，因此根据语法规定，primary key不能再声明在每一个属性声明的地方，而应该是在属性声明完成后再单独声明哪些属性是主码。

模式(SCHEMA)和表(TABLE)

​ 创建基本表(或其他数据库对象)时，如果没有指定模式，那么系统会根据搜索路径来确定该对象所属的模式。另外，RDBMS(数据库管理系统)会使用模式列表中第一个存在的模式作为数据库对象的模式名。

​ 如果搜索路径中的模式名都不存在，系统将给出错误。

SHOW search_path; /* 显示当前的搜索路径 */

修改基本表

​ SQL语句中，修改表的一般语法如下所示。

ALTER TABLE <表名>
    [ADD <新列名><数据类型>[完整性约束] ]
    [DROP <完整性约束名>]
    [ALTER COLUMN <列名><数据类型>];

​ 为了进一步加深理解，举一个对Student表进行修改的例子。现在要向Student表增加“入学时间”列，其数据类型为日期型：

alter table Student add S_entrance date;

​ 需要注意的是，无论基本表中原来是否已经有数据，新增加的列一律为空值。再举一个修改列属性的例子，将年龄的数据类型由字符型改为整数：

alter table Student alter column Sage int;

​ 增加课程名称必须取唯一值的约束条件：

alter table Course add unique(Cname);

删除基本表

​ 删除基本表的一般语法与上文的删除模式基本相同。其中的关键字RESTRICT和CASCADE的用法也和删除模式时相同。

DROP TABLE <表名> [RESTRICT | CASCADE];
