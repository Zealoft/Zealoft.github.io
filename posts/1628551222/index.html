在Windows下使用SUMO进行交通仿真（1）

发表于 2019-03-24 | 更新于 2019-04-11 | 分类于 无人驾驶 | 阅读次数：

写在前面

​ 由于最近实验室有进行交通仿真的任务，需要导入Opendrive地图路网信息并且进行交通仿真，并配合Python API进行通信，起初老师指定使用Vissim进行仿真，但由于学生版的Visssim不能使用Script写Python脚本，旧版本又不能支持Opendrive导入，于是又在网上发现了开源的交通仿真软件SUMO，但这个软件大部分的操作都是在命令行下完成，因此在这里记录一下各种操作。

SUMO在Windows下的安装

​ 安装过程还是比较简单的，但之前花了比较多的时间下载了源码并尝试编译，遇到很多坑后才发现官方提供了Windows版本的binary version，下载使用即可。

​ https://sourceforge.net/projects/sumo/files/sumo/version%200.19.0/sumo-winbin-0.19.0.zip/download

​ 下载完成后解压，并且把解压出目录中的bin目录添加到环境变量中的Path变量即可。

SUMO仿真过程实例

​ 此处以从OpenStreetMap导出地图为例，展示需要的工作流程。

Step 1：获取OpenStreetMap地图

​ 此处不过多赘述，打开OpenStreetMap官网https://www.openstreetmap.org/找到需要的区域后导出osm格式的地图即可。需要注意的是选择的区域大小有限制，不能过大。

Step 2：保存SUMO对OSM文件的预处理代码

​ 根据官网所述，下载的osm文件除了路网信息之外还包含其他信息，需要做简单的处理。

​ OSM-data not only contains the road network but also a wide range of additional polygons such as buildings and rivers. These polygons can be imported using POLYCONVERT and then added to a sumo-gui-configuration.

​ 导出的osm地图不仅包含路网信息也包含大量的别的模块例如建筑和河流，这些模块可以用polyconvert工具添加到sumo-gui配置文件中。

​ 因此，我们手动打开sumo\doc\userdoc\Networks\Import\OpenStreetMap.html文件，然后在打开的网页中间部分 Importing additional Polygons (Buildings, Water, etc.)的下面有一大段xml的代码，复制这些代码，保存为typemap.xml文件。在这里直接将代码复制过来，具体代码见文章最后。

Step 3：准备仿真所需的文件

1.配置SUMO_TOOLS环境变量

​ 安装SUMO完成后，我们需要将SUMO提供的工具包路径加入环境变量。

2.生成路网文件

​ 我们利用SUMO提供的netconvert工具和下载好的osm地图文件生成.net.xml的路网文件。

    netconvert --osm-files map.osm -o map.net.xml

3.生成poly文件

​ 在这一步里，我们利用路网文件、osm地图文件和之前已经准备好的typemap.xml文件生成poly文件。

    polyconvert --net-file map.net.xml --osm-files map.osm --type-file typemap.xml -o map.poly.xml

4.生成旅程和路线文件

​ 路线文件的生成实质上是借助sumo/tools中的randomTrips.py，根据我们已经得到的路网文件来随机生成路线信息，这样就不需要手工设定路线。

    $ python "%SUMO_TOOLS%/randomTrips.py" -n map.net.xml -e 100 -l -o map.trip.xml

​ -n map.net.xml表示输入，-n表述输入的类型是net类型，其实-n等价于–net，-n是一种简写的方式。后面的-l -e 600是随机工具的配置，就像随机数生成函数需要一个种子一样。

​ 这一步执行完成后，得到了一个旅程文件，但实际上我们需要的是.rou.xml的路线文件，此时需要借助duarouter.exe进行操作。但我们并不需要直接运行duarouter.exe，而是再次运行randomTrips.py。

    $ python "%SUMO_TOOLS%/randomTrips.py" -n map.net.xml -r map.rou.xml -e 100 -l
    calling  C:\Program Files (x86)\Eclipse\Sumo\bin\duarouter -n tongji.net.xml -r trips.trips.xml -o tongji.rou.xml --ignore-errors --begin 0 --end 100.0 --no-step-log --no-warnings
    Success.

​ 可以看到python程序自动帮我们调用了duarouter.exe。

5.编辑配置文件

​ 新建一个名为test.sumocfg的文件，在该文件中键入如下的代码：

<configuration>
    <input>
        <net-file value="tongji.net.xml"/>
        <route-files value="tongji.rou.xml"/>
        <additional-files value="tongji.poly.xml"/>
        <junction-files value=""/>
    </input>

    <output>
      <netstate-dump value="tongji.sumo.tr"/>
      <tripinfo-output value="output-tripinfos.xml"/>
      <vehroute-output value="output-vehroutes.xml"/>
    </output>

    <time>
      <begin value="0"/>
      <end value="10000"/>
    </time>

</configuration>

​ 分别将所需的文件填入其中修改为对应的文件名即可。

Step 4：开始进行仿真

​ 有了上述的这些文件，我们就可以利用.sumocfg文件进行仿真了。仿真的方法有很多种，包括使用python代码调用仿真程序、手动进行仿真等，在这里也不多赘述。
