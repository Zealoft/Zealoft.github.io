Linux常用命令的整理

发表于 2019-03-17 | 更新于 2019-03-18 | 分类于 Linux | 阅读次数：

前言

​ 这篇blog主要是写给正在苦学Linux的titi同学，帮助她整理一下平时可能会用到的一些Linux命令，也作为我们两个使用过程中查阅使用。

有哪些基本命令？

关于目录/文件的各种命令

cd命令

​ 首先，用cd命令从当前路径前往指定的目录。

cd [DirectoryName]   # 前往指定的目录

​ 有几种特殊的用法：

cd ~                 # 前往用户的home目录
cd /                 # 前往系统的根目录
cd ..                 # 前往上一层目录

​ 组合使用，就可以任意前往各个层级的各个目录。

ls命令

​ ls命令的用途是列出当前（指定）目录下的所有文件，有很多选项可用。这里列出常用的几种用法，其余的可以需要时再做查询。

​ -l 除了文件名外，还将文件的权限、所有者、文件大小等信息详细列出来。

​ -R -recursive 同时列出所有子目录层

​ -a -all 列出目录下的所有文件，包括以.开头的隐含文件。

​ 一般比较常用的是 ls和ls -al或ls -l。

mkdir命令/touch命令

​ 顾名思义，mkdir命令用作创建目录。而touch命令的作用则是创建指定名称的文件。

rm命令

​ rm命令的作用是删除指定的文件。如果不加任何参数使用时想要删除非空的目录，则会提示错误。因此，实际情况下常用的是rm -rf命令，表示同时删除其子目录（-r），并且不进行确认（-f）。

​ 当然，我们使用-rf选项时需要慎重，确认删除的位置无误后再执行命令，不然一不小心敲出rm -rf /*这样的命令，可就不好了(；′⌒`)

diff命令

​ diff命令的作用是比较两个文件的差异。我们一般常用diff -s来比较两个文件的差异，-s表示两个文件相同的情况下也会输出文件相同的提示。

​ 这个命令是一个非常实用的命令，很多的场合都会有比较文件相同的需求，这时一个diff -s [filename1] [filename2]就能轻松解决问题。

死机了怎么办？

用kill杀死无响应进程

​ 一般来说，使用Linux的过程中如果遇到死机的问题，最简单直接的方法就是使用kill命令杀死无响应的进程。当然，这么做的前提是需要知道进程的名称或是进程号，开启另一个终端窗口使用kill命令结束进程或是kill -9命令强制杀死进程。

​ 在知道进程号pid的情况下，直接使用kill命令即可。

kill -9 [pid]

​ 如果知道的是进程的名称，那么可以首先用ps命令查找进程，得到进程的pid后再使用kill命令。其中，ps的选项-ef表示显示所有进程的详细信息，如果不加-ef选项则表示仅显示运行在前台的进程的信息。

ps -ef | grep [进程名]
kill -9 [pid]

强制重启Linux

​ 但在有些情况下，可能会出现更严重的问题，也就是说Linux系统出现致命错误需要进行重启，并且通过常规的

systemctl reboot

​ 等命令无法正常重启（卡在驱动程序里等等情况），这时候就需要使用特殊的方法进行重启。这些方法会跳过umount文件系统及sync等操作，可能会导致数据损坏，也需要在root用户下使用。

magic SysRq key方法

​ magic SysRq key 通过 proc 接口提供用户直接发底层命令给 kernel 的功能，可以实现关机、重启、宕机等操作，Linux kernel 需要开启 CONFIG_MAGIC_SYSRQ 才可以支持 magic SysRq key。

​ 强制重新启动系统：

# echo 1 > /proc/sys/kernel/sysrq
# echo b > /proc/sysrq-trigger

​ 强制关机

# echo 1 > /proc/sys/kernel/sysrq
# echo o > /proc/sysrq-trigger

​

​ 当然，还有一些其他的强制重启系统的方法，但是只在此处列出用过的方法。
